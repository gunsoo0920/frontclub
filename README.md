- 브렌치 안내
  https://github.com/gunsoo0920/frontclub
  깃허브 들어가시면 각자 이름대로 브렌치를 생성해놨습니다.

* gunsoo : 건수
* jeongeek : 정익
* soongyu : 순규
* sunghyun : 성현

- 먼저 파일 받으시고 npm install 이후에

1. npm run server (프로젝트 실행 명령어),
2. npm run dev (DB.json 실행 명령어)
   순서대로 두가지 다 하셔야 DB문제 없이 실행 가능하십니다요 (혹시나 실행이 안된다면 cd main으로 main 폴더에 위치해 있는지 확인해주세요)

- DB관련 안내
  [books 테이블 구조]

* id : 책 데이터 식별 고유 번호
* category : 책 장르
* title : 책 제목
* isbn : 국제 표준 도서번호
* authors : 책 저자
* publisher : 책 출판사
* image_path : 책 표지 썸네일 URL

-- 그리고 DB.json 파일 코드 맨 밑에
users, reviews 테이블을 미리 만들어 놨습니다. 여기서 해당자분들 사용하시면 될 것 같아요

- 장르 구간 안내
  [id 번호 기준입니다.]

* 1 ~ 25 : 'DEV' / 개발, 프로그래밍 장르
* 26 ~ 50 : 'AI' / 인공지능, 데이터 장르
* 51 ~ 75 : 'SELF' / 자기계발, 공부법 장르
* 76 ~ 100 : 'NOVEL' / 소설, 에세이 장르
* 101 ~ 125 : 'ESSAY' / 인문, 에세이 장르
* 126 ~ 150 : 'BIZ' / 경제, 경영, 스타트업 장르
* 151 ~ 175 : 'KIDS' / 유아, 아동 장르
* 176 ~ 200 : 'HOBBY' / 취미, 라이프 스타일 장르

-- 카카오API가 책을 이름으로 검색해야 하다보니 비슷한 장르끼리 검색해서 최대한 책 이름이 다양하게 나오게 하려고, 딱 하나의 장르가 아닌 비슷한 계열끼리 장르가 뭉쳐있습니다. '대충 이런 느낌의 책들이구나' 라고 생각하시고, 사용하시면 될 것 같아요.

- 파일 구조 안내
  제가 올린 깃 허브 폴더를 보시면 src 폴더 안에
  pages, components, css 폴더를 만들어 놨습니다.
  하여 main.jsx에서 코딩을 하여 파일을 실행시키는게 아닌, pages에서 jsx파일을 생성하고 main.jsx에서는 호출만 해서 작업 진행 현황을 확인해주시면 돼요

* 각각 파일들을 생성할 때 파일 이름 앞에
  메인 : Home
  목록 : Book
  서평 : Review
  로그인 : Login
  해당 이름들을 붙여서 파일을 생성해주시면 감사하겠습니다.
